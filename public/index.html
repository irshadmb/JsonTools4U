JSON Converter Service

Transform, validate, and query your JSON data with powerful tools

Key Features

  • Double JSON string parsing
  • JSON structure validation
  • JSONPath query support
  • YAML to JSON conversion
  • XML to JSON conversion
  • Data Validation Rules Builder
  • Custom validation patterns
  • Multi-field validation
  • Error handling for invalid JSON
  • Simple REST API endpoints
  • Lightweight and fast

How to Use

  1. Send a POST request to /convert, /validate, or /query
  2. Include your JSON in the request body
  3. Add JSONPath expression as query parameter for /query
  4. Receive parsed JSON, validation, or query results

Data Validation Rules Builder

  • Build custom validation rules
  • Support for multiple data types
  • Format validation (email, date, URI)
  • Array and object validation
  • Custom error messages
  • Reusable validation templates

JSON Data Mapper

  • Map data between different JSON structures
  • Support for nested objects and arrays
  • Custom data transformations
  • Flexible path-based mapping
  • Built-in data type conversions

CSV to JSON Converter

  • Convert CSV data to JSON arrays
  • Automatic header detection
  • Type casting support
  • Empty line skipping
  • Trim whitespace

JSON Object Flattening

  • Flatten nested JSON objects
  • Custom delimiter support
  • Array index preservation
  • Deep nesting support
  • Path-based key generation

Example Requests

1. Convert Endpoint

POST /convert
                        Content-Type: text/plain
                        
                        "{\"name\":\"John\",\"details\":{\"age\":30,\"city\":\"New York\"}}"

2. Validate Endpoint

POST /validate
            Content-Type: application/json
            
            {
                "name": "John",
                "age": 30,
                "email": "john@example.com"
            }

3. JSONPath Query Endpoint

POST /query?path=$.store.book[*].author
            Content-Type: application/json
            
            {
                "store": {
                    "book": [
                        {
                            "author": "John Smith",
                            "title": "Sample Book"
                        }
                    ]
                }
            }

4. YAML to JSON Endpoint

POST /yaml2json
                        Content-Type: text/plain
                        
                        user:
                          name: John Doe
                          age: 30
                          address:
                            street: 123 Main St
                            city: New York
                            country: USA
                          hobbies:
                            - reading
                            - gaming
                            - traveling

5. XML to JSON Endpoint

POST /xml2json
                        Content-Type: text/plain
                        
                        <?xml version="1.0" encoding="UTF-8"?>
                        <user>
                            <name>John Doe</name>
                            <age>30</age>
                            <address>
                                <street>123 Main St</street>
                                <city>New York</city>
                                <country>USA</country>
                            </address>
                            <hobbies>
                                <hobby>reading</hobby>
                                <hobby>gaming</hobby>
                                <hobby>traveling</hobby>
                            </hobbies>
                        </user>

6. Build Validation Rule

POST /build-validation-rule
                        Content-Type: application/json
                        
                        {
                            "fieldName": "email",
                            "ruleType": "string",
                            "parameters": {
                                "required": true,
                                "format": "email",
                                "minLength": 5,
                                "maxLength": 100
                            }
                        }

7. Validate With Rules

POST /validate-with-rules
Content-Type: application/json

{
    "data": {
        "email": "user@example.com",
        "age": 25
    },
    "rules": [
        {
            "field": "email",
            "type": "string",
            "required": true,
            "format": "email"
        },
        {
            "field": "age",
            "type": "number",
            "required": true,
            "minimum": 18,
            "maximum": 100
        }
    ]
}

8. JSON Data Mapping

POST /map-json
Content-Type: application/json

{
    "sourceData": {
        "user": {
            "firstName": "John",
            "lastName": "Doe",
            "contact": {
                "email": "john.doe@example.com"
            }
        }
    },
    "targetStructure": {
        "person": {
            "fullName": "",
            "email": ""
        }
    },
    "mappingRules": [
        {
            "sourcePath": "user.firstName",
            "targetPath": "person.fullName",
            "transform": {
                "type": "custom",
                "function": "return value + ' ' + sourceData.user.lastName"
            }
        },
        {
            "sourcePath": "user.contact.email",
            "targetPath": "person.email"
        }
    ]
}

9. CSV to JSON Endpoint

POST /csv2json
Content-Type: text/plain

name,age,email,city
John Doe,30,john@example.com,New York
Jane Smith,25,jane@example.com,Los Angeles
Bob Johnson,35,bob@example.com,Chicago

10. Flatten Nested JSON

POST /flatten
Content-Type: application/json

{
    "data": {
        "name": "John Doe",
        "age": 30,
        "address": {
            "street": "123 Main St",
            "city": "New York",
            "country": "USA"
        },
        "contacts": [
            {
                "type": "email",
                "value": "john@example.com"
            },
            {
                "type": "phone",
                "value": "1234567890"
            }
        ]
    },
    "delimiter": "."
}
